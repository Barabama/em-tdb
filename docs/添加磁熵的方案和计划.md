# 添加磁熵的方案和计划

## 一、背景与必要性

CoCrNiV 体系中含有强磁性元素（Co, Fe, Cr, Ni），忽略磁熵会导致高温下的吉布斯自由能严重偏离实验值，进而导致占位分数预测错误。

## 二、核心方案

### 1. 磁熵计算模型

采用 CALPHAD 标准的热力学表达式，将吉布斯自由能分为两部分：

$$ G_{total}(T) = G_{phvibelec}(T) + G_{mag}(T) $$

其中：
- $G_{phvibelec}(T)$：现有 Phonopy-QHA 输出的吉布斯自由能曲线
- $G_{mag}(T)$：使用 Inden (1976) 模型计算的磁贡献

### 2. 所需磁学参数

对于每个含磁端基，需要获取：
- **居里温度 ($T_c$)**：K，可查 SGTE 数据库或实验值
- **玻尔磁子数 ($n_B$)**：每个原子的磁矩大小
- **温度缩放因子 ($b$)**：通常取 0.5 (Fe/Ni/Co)

### 3. 数学模型

$$ \Delta G_{mag} = RT \ln(n_B + 1) \cdot f(\tau) $$

其中 $\tau = T / T_c$，$f(\tau)$ 是德拜积分形式的函数。

## 三、具体操作流程

### 1. 数据处理

无需重新运行 VASP，仅在**后处理阶段**修改生成的 `gibbs-temperature.dat` 数据：

1. 编写 Python 脚本（如 `fix_mag_tdb.py`）处理 Phonopy 输出
2. 加载 `gibbs-temperature.dat` 数据
3. 定义磁学参数（示例值）：
   ```python
   mag_params = {
       'Co': {'Tc': 1394.0, 'nB': 1.66}, 
       'Fe': {'Tc': 1043.0, 'nB': 2.2}, 
       'Ni': {'Tc': 627.0, 'nB': 0.60}
   }
   ```
4. 计算有效 $T_c$ 和 $n_B$（可采用加权平均或参考文献值）
5. 叠加磁项到总自由能
6. 保存修正后的数据：`gibbs_temperature_magnetic_corrected.dat`

### 2. 多项式拟合

将修正后的 $G_{total}(T)$ 散点图进行 POLYNOMIAL 拟合，保留系数 A, B, C...

### 3. TDB 文件写入

直接将磁熵贡献拟合进 $G-T$ 的多项式系数中，确保兼容性：

```text
FUNCTION G_FeNi3_FCC  ! 名称
  ! 振动部分 (来自 QHA 拟合系数) + 磁学部分
  -100000 -15*T*LN(T) -0.001*T**2 + ...
END
```

## 四、行动计划

### 近期任务（1-2周）

1. **脚本开发**：编写 `fix_mag_tdb.py` 脚本，实现磁熵计算和数据修正
2. **参数收集**：收集 Co, Cr, Fe, Ni, Mn 的 $T_c$ 和磁矩数据（SGTE 数据库或文献）
3. **数据处理**：处理现有 `gibbs-temperature.dat` 文件，生成修正后的自由能数据
4. **多项式拟合**：对修正后的数据进行多项式拟合，获取新的系数
5. **TDB 更新**：更新 TDB 文件，包含磁熵贡献

### 中期任务（2-4周）

1. **数据库验证**：用 Pandat 或 Thermo-Calc 验证 TDB 文件，运行二元相图测试
2. **过剩自由能处理**：（可选但推荐）补充 SQS 混合焓计算，拟合固溶体相的 Redlich-Kister 参数
3. **论文修改**：重写 Introduction 和 Methods 章节，明确声明包含磁学贡献

## 五、预期成果

- 修正后的热力学数据库，包含磁熵贡献
- 更准确的高温吉布斯自由能预测
- 更可靠的占位分数计算结果
- 回应审稿人关于磁熵和过剩自由能的质疑

## 六、注意事项

- 复合端基的有效 $T_c$ 计算需参考文献或采用经验公式
- 确保磁学参数的准确性，建议优先使用 SGTE 数据库值
- 保持与现有 DFT+QHA 流程的兼容性，仅在数据处理阶段添加磁熵修正
